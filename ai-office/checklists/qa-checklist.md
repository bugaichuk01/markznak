# QA чек-лист (Tester)

Запускать **на каждом** handoff от программиста перед передачей Team Lead.

**Объём проверки** берётся из **постановки (Acceptance Criteria)** и **типа доставки** (сервис в Docker, десктоп-клиент, только доки и т.д.). Не все пункты ниже обязательны для каждой задачи — отметь **только применимое** и укажи в отчёте, что проверялось.

---

## Универсально

- [ ] **Acceptance Criteria** из постановки проверены по пунктам.
- [ ] Секретов в репозитории нет.
- [ ] README / docs обновлены при изменении поведения или установки (если в задаче менялось поведение или способ запуска).

---

## Сборка и запуск (если применимо)

- [ ] **Build:** шаг сборки завершился успешно (`docker compose build`, `npm run build`, `poetry install` / `pip install` и т.п. — **что указано в спеке**).
- [ ] **Run:** шаг запуска завершился успешно (сервис стартует, скрипт отрабатывает — **что указано в спеке**); нет необоснованного падения при старте.

Если в задаче **нет** сборки/рантайма (например только markdown, только правка конфига без сервиса) — эти пункты **не применяются**; напиши в отчёте «N/A» с кратким обоснованием.

---

## Платформенная проверка

Отметь секции, которые относятся к задаче.

### Python service (backend, скрипты с зависимостями)

- [ ] Зависимости и запуск по README / `pyproject.toml` / спеке; при наличии сервиса — смок основного пути (API, CLI).
- [ ] При наличии **`.env`**: `.env.example` обновлён при новых переменных.
- [ ] Важные действия логируются; нет «тихих» провалов без причины (если в спеке ожидается логирование).

### Frontend (React/Vite/TS и т.п.)

- [ ] Сборка/дев-сервер по README; UI открывается; ключевой сценарий из постановки работает.

### Десктоп-клиент (например Windows/Python GUI)

- [ ] Сборка/запуск по инструкции клиента; ключевой сценарий (запись/загрузка и т.д.) работает согласно спеке.

### Только документация / конфиг без сервиса

- [ ] Изменения согласованы со спекой; ссылки и пути корректны; нет утечки секретов в тексте.
- [ ] Если менялся конфиг — проверена согласованность с остальным репо и инструкциями (README).
